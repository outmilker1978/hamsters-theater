**Версия продукта:** 1.7.1  
**Дата:** июнь 2026

---

## 1. О продукте

TV Hamsters — десктопное приложение для совместного просмотра видео с видеочатом,
демонстрацией экрана и передачей системного звука.

Работает как через локальную сеть (LAN), так и через Интернет (через облачный сервер на Render.com).

## 3. Архитектура

### Локальный режим
На ПК хоста запускается встроенный Socket.IO сервер (порт 3000+).
Гость подключается напрямую по IP:порт. UPnP пытается открыть порт.

### Облачный режим
Оба ПК подключаются к серверу на Render.com. Сервер только пересылает
сигналы (offer/answer/ICE). Видео/аудио — напрямую P2P через WebRTC + STUN.

```
ПК1 ←—— WebRTC (P2P) ——→ ПК2
  │                         │
  └── Socket.IO ——→ Render.com ——→┘
```

## 5. Облачный сервер (Render.com)

**Адрес:** https://hamsters-theater-cloud.onrender.com
**Репозиторий:** https://github.com/outmilker1978/hamsters-theater-cloud

### Обрабатываемые события

| Событие | Назначение |
|---------|-----------|
| `create-room` / `room-created` | Создание комнаты (4-значный PIN) |
| `join-room` / `joined` / `peer-joined` | Подключение к комнате |
| `offer` / `answer` | Обмен SDP (WebRTC) |
| `ice-candidate` | ICE кандидаты |
| `signal` | Прикладные сигналы (screen, PTT) |
| `disconnect` | Очистка при отключении |

### Как управлять

1. Зайти на https://dashboard.render.com (войти через GitHub)
2. Выбрать Web Service `hamsters-theater-cloud`
3. Доступно:
   - **Logs** — логи сервера
   - **Manual Deploy → Deploy** — перезапуск
   - **Auto-Deploy** — автообновление из GitHub

### Важно

Render бесплатный — сервер засыпает после 15 мин бездействия.
При подключении просыпается за 5-15 сек. Для постоянной работы — $7/мес.

### Как обновить URL

URL задаётся в двух файлах:
- `main.js` — константа `CLOUD_SERVER_URL`
- `renderer/app.js` — начальное значение `CLOUD_SERVER_URL`

После изменения — пересобрать .exe.

## 6. Сборка .exe

**Требования:** Node.js v24, npm 11, зависимости установлены (`npm install`)

```bash
# Упаковка app.asar
npm run build

# Портативный .exe (из ht-win)
npx electron-builder --prepackaged "C:\Users\Hamster\AppData\Local\Temp\ht-win" --win portable
```

**Важно:** перед сборкой удалить кэш `dist\*.nsis.7z`, если менялось содержимое prepackaged.

Ошибка `winCodeSign` про симлинки — НЕ критична.

Результат: `dist\TV Hamsters X.X.X.exe`

**Важно:** Windows Defender может блокировать старый .exe —
используйте новый номер версии в имени файла.

## 7. Разработка

### Запуск для отладки
```bash
npm start          # запуск Electron
F12                # DevTools
```

### Версионирование
Перед сборкой обновить версию в:
1. `package.json` — `"version": "X.X.X"`
2. `main.js` — `title: 'TV Hamsters (vX.X.X)'`
3. `renderer/index.html` — три места: `vX.X.X`

### Архитектура multi-user
- **Mesh-топология:** каждый клиент создаёт отдельный RTCPeerConnection для каждого участника
- `peers = {}` — карта peerId → { pc, screenPC, remoteStream, cameraCandidates, screenCandidates }
- Новый участник рассылает `user-joined` всем в комнате, каждый существующий создаёт offer
- Screen sharing: один screenPC на каждого пира (транслируется всем при старте)
- PTT мьютит remoteStream конкретного шарера
- Сервер: лимит комнаты повышен до 5

### Обратная совместимость
⚠️ **Обязательное правило:** каждая новая версия должна сохранять совместимость со старыми клиентами. Если это невозможно (например, ломается протокол сигнализации), сервер должен поддерживать оба формата событий, и клиент должен слушать оба варианта. Если совместимость принципиально невозможно сохранить — **сообщи пользователю перед сборкой**.

### Важные моменты
- Изменение `server/cloud.js` → push на GitHub → Render перезапустит сам
- Изменение `main.js` или `renderer/*` → пересобрать .exe
- Формат: семантическое версионирование (major.minor.patch)

## 9. Известные проблемы

- Render бесплатный — засыпает после 15 мин
- UPnP не работает с мобильными хотспотами
- Windows Defender блокирует старый .exe при перезаписи
