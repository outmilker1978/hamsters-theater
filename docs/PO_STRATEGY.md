## 0. Сравнение безопасности (P2P vs облачные сервисы)

| Аспект | TV Hamsters (P2P) | Zoom / Discord |
|--------|------------------|----------------|
| Видео/аудио | Полностью E2E (DTLS-SRTP) | Сервера могут слушать (кроме Zoom E2E режима) |
| Чат | Через сервер (виден Render) | Через сервер (виден Zoom/Discord) |
| Аутентификация | Нет (код комнаты) | Есть (аккаунты) |
| Независимость | Не нужен чужой сервер для медиа | Полная зависимость от платформы |

*Источник: разработчик. Использовать для маркетинга и статей.*

## 2. Аналитика и метрики

### Внедрённые инструменты

| Инструмент | ID | Что отслеживает | Статус |
|-----------|-----|-----------------|--------|
| Yandex.Metrica | `109746304` | Посещения, поведение, цели | ✅ Установлен на сайте |

### Какие метрики отслеживать в первую очередь

**Веб-аналитика (Yandex.Metrica):**
1. **Количество визитов** — общий трафик сайта
2. **Источники трафика** — откуда приходят люди (поиск, соцсети, прямые переходы)
3. **География** — Россия, СНГ, другие страны
4. **Клики по кнопке скачать** — конверсия в загрузку
5. **Отправка формы обратной связи** — конверсия в лид
6. **Переходы на Boosty / CloudTips** — конверсия в донат
7. **Отказы (bounce rate)** — процент ушедших сразу
8. **Глубина просмотра** — сколько секций смотрят

**Продуктовые метрики (нужна реализация):**
1. **Количество установок** — сколько раз скачали .exe
2. **Количество созданных комнат** — через сайт? через бота?
3. **Daily/Monthly Active Users** — нужен телеметрия
4. **Retention** — возвращаются ли пользователи
5. **Referral** — сколько привели друзей

### Рекомендации по доработке аналитики

1. **Цели в Yandex.Metrica:**
   - Цель "Скачал" — клик по кнопке download
   - Цель "Написал в форму" — успешная отправка формы
   - Цель "Клик на Boosty/CloudTips" — переход на донат
2. **Счётчик загрузок:** сделать редирект с сайта на GitHub Release через промежуточную страницу или beacon, чтобы засчитывать загрузки
3. **Telegram-аналитика:** если бот комнат отвечает, считать количество созданных комнат
4. **Для Freemium:** добавить простой heartbeat-пинг (раз в день) от приложения к серверу для DAU

## 7. Принципы принятия решений (для PO)

1. **Сначала пользователь** — любая фича должна приносить ценность пользователю
2. **Минимальными шагами** — не делать "идеально", делать "достаточно хорошо"
3. **Данные важнее мнений** — решения на основе аналитики, а не интуиции
4. **Поэтапная монетизация** — Freemium только после того, как продукт доказал ценность
5. **Прозрачность** — с пользователями честно, никаких скрытых платежей
6. **Сначала RU, потом EN** — российская аудитория как база, английский рынок как масштабирование
7. **Сначала обычные люди, потом бизнес** — product-market fit на массовом пользователе
