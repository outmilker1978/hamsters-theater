# TV Hamsters — Гайд администратора

## Полный цикл релиза

### Подготовка (делаю я — разработчик)
1. Готовлю код, обновляю версию в `package.json`
2. Пушаю в `master`

### Автоматически после пуша
- Render авто-деплоит сервер (`bot/index.js`) — `Auto-Deploy: On Commit`
- GitHub Pages деплоит сайт (если менялись `docs/`) — `Auto-Deploy: On Push`

### Сборка (делаешь ты)
3. Открываешь PowerShell в папке `NetTheater`
4. Запускаешь: `npm run build-portable`
5. Готовый `.exe` лежит в `dist\TV.Hamsters.X.Y.Z.exe`

### Публикация (делаешь ты)
6. Создаёшь GitHub Release:
   - Тег: `v1.x.x` (например `v1.7.5`)
   - Название: краткое описание (например "Название версии")
   - Описание: что нового для пользователей
   - Прикрепляешь `.exe` из `dist/`

### Автоматически после релиза
- Telegram-канал @tvhamsters — пост с release notes ✅
- Сайт — версия, ссылка скачивания обновляются автоматически ✅

### Коммуникация с Оунером (делаю я)
7. Я пишу тебе список изменений для пользователей (human-readable)
8. Ты пересылаешь Оунеру — он обновляет контент сайта (тексты, скриншоты, блог)
